# Sobre mim

Sou **Hendel**, programador e assíduo estudante de **programação com IA**.

Meu foco é entender e testar modelos de linguagem na prática — **OpenAI**, **Claude**, **OpenCode** e **Copilot** — e usar essas ferramentas pra construir sistemas funcionais, com cuidado real com **UI/UX**.

Não fico só teorizando: **testo, quebro, conserto e construo**. A ideia é sair do "achei legal" pra "fiz rodar de verdade".

## O que estou fazendo

- **Experimentos com LLMs** — comparando OpenAI, Claude, modelos locais e agentes como OpenCode e Copilot em tarefas reais de código
- **Testes com linguagens** — provando stacks diferentes pra entender os trade-offs na prática, não no hype
- **Sistemas funcionais** — a meta não é só protótipo jogado fora, é software que serve pra alguém, com interface que não dói usar
- **UI/UX** — interface existe pra ser usada; funcional sem usabilidade é só uma demo

## Como trabalho

Aprendo fazendo. Escrevo o caminho (os acertos e os perrengues) neste blog porque:

1. **Documentar obriga a entender** — se não dá pra explicar, não entendi direito
2. **Deixa rastro** — pra eu mesmo voltar e pra quem chegar depois
3. **Reforça a marca** — mostrar o processo vale mais que mostrar só o resultado

## Sobre o blog

**HendelCode** é meu caderno de laboratório público. Roda **Hugo** com o tema **Hextra**, hospedado de graça no **GitHub Pages**, com build e deploy automáticos via **GitHub Actions**. Tudo em Markdown, versionado no Git.

A inspiração vem de blogs como o [AkitaOnRails](https://akitaonrails.com/) — escrever em Markdown, publicar direto, sem frescura, com identidade visual limpa.

O código fonte é aberto em [github.com/hendelsantos/blog_hendelcode_Hugo](https://github.com/hendelsantos/blog_hendelcode_Hugo). Encontrou erro ou quer sugerir algo? Abre issue.
